# mailverdict

Keyless email intelligence API (disposable/role/free detection, MX via DoH,
typo suggestions). Live at `https://api.mailverdict.dev` (Cloudflare Workers);
npm package in `sdk/` (not yet published).

## Commands

Requires Node >= 20 (uses global fetch, AbortSignal.timeout).

- `npm run build:data` ‚Äî fetch 11 upstream lists, merge -> `data/dataset.json` + `data/meta.json` + `data/changelog.json`. Run before first `npm run dev`; safe to re-run anytime.
- `npm run build:snapshot` ‚Äî regenerate `sdk/src/vendor/` + `sdk/src/snapshot.json` from the canonical logic/dataset. Run after build:data when lib code or dataset changed; commit the results.
- `npm run dev` ‚Äî Node server on :8787 (PORT env to override)
- `npm test` ‚Äî vitest; network-free except the SDK live-path test, which serves the app in-process on an ephemeral port
- `npm run typecheck` ‚Äî covers src, pipeline, test, and sdk
- `npm run build:sdk` ‚Äî compile the SDK package to `sdk/dist`
- `npm run mcp` ‚Äî MCP stdio server (tools: check_email, check_domain)
- `npx wrangler deploy` ‚Äî deploy to production (the daily Action also does this)

## Architecture

- `pipeline/` ‚Äî dataset build. Sources in `sources.ts` (permissive licenses
  only ‚Äî no GPL/unlicensed; record license per source). Merge precedence is
  load-bearing: allowlists rescue from disposable, then **disposable wins over
  free** (the HubSpot free list contains mailinator.com and typo-squats).
  The build also appends per-build domain adds/removes to `changelog.json` ‚Äî
  but only between two fully-clean builds (a failed source would read as false
  churn, which would poison the freshness feed).
- `src/lib/` ‚Äî pure logic, no I/O except `mx.ts` (DNS-over-HTTPS so Node and
  edge runtimes share one code path). `check.ts` composes the public response.
- `src/mcp-http.ts` ‚Äî the hosted remote MCP server: a stateless JSON-RPC
  handler (no Durable Objects, so it runs on the free Worker) wired into
  `app.ts` at `POST /mcp`. Reuses `checkEmail`/`checkDomain`. `src/mcp.ts` is
  the separate stdio server for local use; keep the two tool lists in sync.
- `src/app.ts` ‚Äî Hono app, runtime-agnostic. `worker.ts` (Cloudflare Workers ‚Äî
  production) and `server.ts` (Node ‚Äî self-hosting) both wrap it. The worker
  bundles the dataset and docs at deploy time, so dataset refreshes require a
  redeploy (the daily Action handles this).
- `data/` ‚Äî generated but committed (versioned dataset is part of the product;
  the GitHub Action refreshes it daily).
- `sdk/` ‚Äî the npm package (live API with bundled-snapshot fallback).
  `sdk/src/vendor/` and `sdk/src/snapshot.json` are GENERATED by
  `npm run build:snapshot` ‚Äî never edit them by hand; edit `src/lib/` and
  regenerate.

## Conventions

- Response shape is the Kickbox/Emailable industry convention: `result`
  (deliverable|undeliverable|risky|unknown), `reason` vocabulary,
  bare `disposable`/`role`/`free` booleans, `did_you_mean` = full corrected
  address, `mx_found`/`mx_records`, `normalized_email`, `score` 0-100,
  `accept_all: null` reserved. Don't rename these ‚Äî migration compatibility
  with other providers is a deliberate feature.
- Typo suggestions use Damerau/OSA distance (transpositions cost 1); free-list
  domains get distance-1 budget only, disposable domains never get suggestions.
- Keyless access is core to the product: never add a mandatory API key to v1
  endpoints.
- **Never log email addresses.** Aggregate, PII-free request counts (e.g.
  Cloudflare zone analytics) are fine. Public phrasing must stay precise:
  "we never log email addresses," not "we log nothing."
- Recommend the domain-only check (`/v1/domain/*`) as the default integration
  in docs and examples ‚Äî no email address leaves the caller's system.
